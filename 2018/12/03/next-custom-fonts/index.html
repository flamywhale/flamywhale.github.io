NexT 主题自定义字体

发表于 2018-12-03 | 更新于 2018-12-04 | 分类于 2018年12月
设置主题自定义字体时出现的问题及解决方案

博客环境

  • 博客驱动：Hexo
  • 博客主题：NexT
  • 博客服务器：Github Pages

博客布局

问题

基于上述博客结构，我对 themes/next/_config.yml 中的字体设置进行了如下修改：

  • 将全局字体<global>设置为<Lato>
  • 将代码块字体<code>设置为<Source Code Pro>

在使用命令 hexo s --debug 进行本地测试时，页面字体加载成功。

但是，我将博客生成静态页面部署至我的 github repo 上后，再访问 flamywhale.github.io 时，发现字体并没有成功替换。

问题原因

打开浏览器开发者选项，通过抓包发现，NexT 的字体设置是调用Google字体的 API （fonts.googleapis.com）来加载字体的 css ，但是此时API的请求收到了 403 ERROR 的回应。由此判断，此时的github服务器出现了一些问题。

因此，为了防止调用谷歌字体API失败，使得预设字体无法正常加载，我决定将字体文件和对应的 css 在本地进行设置。这样就能够在生成静态网页时，将字体文件加载入网页的 css 中，而不用再调用谷歌字体的 API 来获取字体文件和对应的 css 文件了。

解决方案

下载开源字体

  1. 访问谷歌字体，找到对应字体。
  2. 通过该字体调用的 url 找到其 css 设置，然后下载 woff2 格式的文件到本地。
  3. 将字体的 woff2 文件保存至主题目录 themes/next 下的 source/fonts 文件夹中。
